How to Use This Application

This is a low-latency, deterministic trade compliance router. It ingests a raw trade order, routes it through regulatory checks, and streams each internal graph-state transition back in real time. No LLM and no database are involved — every decision is explicit, programmatic, and runs entirely in-memory.

How an order flows

ingest→ crypto_compliance/ equity_compliance/ dead_letter→ APPROVED / BLOCKED
  • Crypto → crypto_compliance: BLOCKED if notional > $100,000 (volatility limit), else APPROVED.
  • Equity → equity_compliance: APPROVED only if the symbol is in the in-memory market universe.
  • Anything else → dead_letter: automatically BLOCKED (fail-safe).

Step 1 — Route a single order

  1. Go to the Route Order tab.
  2. Fill in the form, or click a preset (e.g. Crypto Block, Dead-letter).
  3. Press Route Order ▸ and watch each node appear live with its status badge, reason, and timestamp.

Step 2 — Run the test suite

  1. Go to the Test Suite tab.
  2. Press Run all tests ▸. Each of the 8 scenarios is sent to the API and checked against its expected outcome.
  3. Review the PASS/FAIL grid, the latency per order, and click view raw on any row to see the exact JSON the backend returned.

The volatility boundary is exclusive: a crypto notional of exactly $100,000 is APPROVED — only strictly greater is blocked.

Call the API directly

Route one order and get the final JSON result:

curl -X POST http://127.0.0.1:8000/api/orders \
  -H 'Content-Type: application/json' \
  -d '{"order_id":"C2","asset_class":"crypto","symbol":"BTC","quantity":5,"price":50000}'

Stream the live state transitions as Server-Sent Events:

curl -N -X POST http://127.0.0.1:8000/api/orders/stream \
  -H 'Content-Type: application/json' \
  -d '{"order_id":"E1","asset_class":"equity","symbol":"AAPL","quantity":100,"price":180}'

Endpoints: GET /api/config · POST /api/orders · POST /api/orders/stream.
